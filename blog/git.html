January 4th 2016

1. What are the benefits of version control? There are many benefits of using a version control system (VCS) such as

  • Collaboration: Without VCS you and/or team of programmers are working on a set files all together in a same file! [Scary]
  • Storing: It will let you to get to a habit of storing your code consistently in a proper manner.
  • Restoring: In case you have done some unneccsary changes, you can always go back and redo.
  • Documentaion: Everytime you commit to changes you have to add a note on what is changing

    • 2. How does git help you keep track of changes? Git will use a "HEAD" index to point to your files inside a place created by user which called repository. These are the possible stages in any repository:

  • Untracked: This is where a file in a repository is modified but not yet added to git to track changes. Run Command "Git add 'Name_Of_The_File'"
  • Unmodified: This is where the file has in tracking stage and has not yet been changed by user
  • Modified: This is where the file is in tracking stage and has been modified and is ready to commit
  • Staged: This is where the file is added to stage

    • 3. Why use GitHub to store your code? In general using git will be stored locally on your computer so in case of data corruption, everythig WOULD BE GONE! Using Github will store your data remotely, in my opinion these are the advantages of using Github:

  • Having more flexibility over Git
  • You can share your data with the world or keep private
  • Stable Cloud
  • Although not recommended but you can use the app for easier UI
